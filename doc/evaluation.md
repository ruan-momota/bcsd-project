benchmark:
- query: 1000, pool: 100000，取自['clamav', 'curl']两个项目
- 过滤脏数据后，clamav有83093个函数，curl有37620，共计120713
- query选取：随机选取1000个不同函数名的函数
- pool选取：除去query的函数，将1000个query对应函数名的所有variants加入pool
再用剩下的其余函数凑足100000

evaluation:
- 1. 输入(1000, 100000)的相似度矩阵
- 2. 将相似度矩阵每一行的得分换成索引，并按照原始分数排序
[[ 0.2,    0.9,    0.5 ],     # query 0
 [ 0.8,    0.1,    0.3 ]]     # query 1
换成索引 ->
[[ 0,    1,    2 ],   
 [ 0,    1,    2 ]]
按照原始分数排序 -> 
[[ 1,    2,    0 ],   
 [ 0,    2,    1 ]]
- 3. 计算recall@1：看每一行[0]的位置是否是正确答案
如果是，count+=1,最后recall@1=count/query总数

- 4. 计算mrr@10：遍历每行，看前10个是否正确，第一个正确就sum+=1,第10个正确就sum+= 1/10
最后计算sum/query总数
